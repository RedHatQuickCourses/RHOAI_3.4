Automate RAG Optimization with AutoRAG (Technology Preview)

What is it?

AutoRAG is an automated workflow that identifies the optimal Retrieval-Augmented Generation (RAG) configuration for your specific data. It automates the process of testing variations in the Search Space—including chunk size, embedding models, vector stores, and top-K retrieval settings—using an automated optimization pipeline.

Why does it matter?

Building a high-quality RAG system manually is a slow, iterative process involving expert-level guesswork. AutoRAG shifts this burden from data scientists to automated pipelines, providing:

  • Hyper-parameter Optimization: Systematically tests multiple RAG algorithms and techniques against your specific use case.

  • Metric-Driven Selection: Evaluates each configuration using advanced metrics such as Answer Correctness and Faithfulness to ensure generated responses are grounded and accurate.

  • Accelerated Time-to-Market: Moves from raw documents to a validated, high-performing RAG configuration in minutes instead of days.

How to use it?

  • Prepare Test Data: Gather your source documents and a set of test Question-Answer (QA) pairs.

  • Create Optimization Run: Use the RHOAI dashboard or Jupyter Notebooks to define your search space and trigger the optimization run.

  • Evaluate Results: Review the Leaderboard to compare performance across different configurations.

  • Deploy: For top-performing configurations, AutoRAG generates a Jupyter Notebook that you can use to immediately deploy and run the optimized RAG pattern.
